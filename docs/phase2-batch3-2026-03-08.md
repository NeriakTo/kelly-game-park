1. AI 呼叫逾時控制

## 已完成變更

### 1) AI 逾時控制（`src/services/ai.ts`）
- 新增 `AI_TIMEOUT_MS = 6000`
- OpenAI / Gemini 呼叫統一走 `fetchWithTimeout`
- 逾時會回傳 `reason: 'provider_timeout'`，並自動 fallback 本地題庫

### 2) schema 驗證與正規化強化
- `isMathProblem`：
  - 必要欄位皆為非空字串
  - `answer` 必須是有限數值
- `sanitizeMathProblem`：
  - 強制使用 request 的 `unitId`（避免 AI 跨單元漂移）
- `isShopQuestion`：
  - `description` / `hint` 必填
  - `options` 若存在需 2~6 個有效數字
- `sanitizeShopQuestion`：
  - options 去重、過濾非法值
  - 保證 `answer` 一定存在於 options（若 options 存在）

### 3) Smoke 三情境檢核建議
- Case A：無 API Key → 應顯示本地題庫，遊戲可正常進行
- Case B：有效 API Key → 可看到 AI 題目來源
- Case C：故意填錯 Key / 斷網 → 應自動 fallback 並顯示 AI 暫不可用
